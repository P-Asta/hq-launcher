# 2. 문제 작성 기능
문제 제작자는 가장 먼저 다음 정보를 입력한다.

* 문제 제목
* 문제 설명

문제 설명은 Markdown 문법을 지원한다.

또한 상단 툴바를 통해 다음 기능을 사용할 수 있다.

* Bold
* Italic
* Underline
* Code Block
* Inline Code
* Link
* List
* Heading
* Quote
* Table
* 수식 입력

사용자는 글자를 선택한 뒤 버튼을 눌러 스타일을 적용할 수 있다.

문제 설명은 실시간 미리보기를 지원한다.


# 3.1 테스트케이스 관리

문제 제작자는 실제 채점에 사용되는 테스트케이스를 관리할 수 있다.

각 테스트케이스는 다음 정보를 가진다.

* 입력 데이터
* 출력 데이터

제공 기능:

* 테스트케이스 직접 작성
* 테스트케이스 수정/삭제
* 테스트케이스 복사
* 다중 선택 관리
* ZIP 다운로드
* ZIP 업로드
* 기존 테스트케이스 덮어쓰기
* 테스트케이스 추가 업로드
* 테스트케이스 미리보기

ZIP 업로드 시 자동으로 테스트 파일 구조를 분석한다.

예시:

```text
0.metadata
1.in
1.out
2.in
2.out
```


# 3.2 AI 테스트케이스 생성 기능

특정 플랜 이상의 유저는 AI 기반 테스트케이스 생성 기능을 사용할 수 있다.

생성된 테스트케이스는 자동 적용되지 않으며, 제작자가 검토 후 선택적으로 추가할 수 있다.


# 4 공개 예시 입력/출력

문제 설명 작성 후에는 공개 예시 입력/출력을 설정할 수 있다.

이전에 설정한 테스트케이스를 선택할수 있는 select menu에서 공개할 테스트케이스들을 선택할수있다.

선택한 테스트 케이스는 아래에 표시되며, 추가 설명을 작성할수 있다.

공개할 테스트케이스 리스트와, 추가 설명 정보는 54번 라인에 써있는 0.metadata파일에 저장된다

* 예시 입력
* 예시 출력
* 추가 설명 (선택)

제작자는 다음 기능을 사용할 수 있다.

* 공개 여부 설정
* 공개 순서 변경
* 설명 추가
* 공개 상태 미리보기

예시:

| 테스트 번호 | 공개 여부 |
| ------ | ----- |
| #1     | 공개    |
| #2     | 공개    |
| #3     | 비공개   |
| #4     | 비공개   |



# 5. 문제 메타데이터 설정

문제 작성 마지막 단계에서 제작자는 문제 정보를 설정한다.

입력 항목:

* 문제 태그
* 문제 난이도
* 제한 시간
* 메모리 제한
* 공개 범위
* 언어 제한 (선택)

# 6. 저장 및 업로드

제작자는 다음 기능을 사용할 수 있다.

* 임시 저장
* 사본 저장
* 미리보기
* 업로드

## 6.1 사본 저장

현재 문제를 새로운 Draft로 복제한다.

## 6.2 업로드

문제를 검토 상태로 업로드한다.

업로드 후에는 일반 공개되지 않는다.


# 7. 문제 상태 시스템

문제는 다음 상태를 가진다.
```
| 상태                | 설명    |
| ----------------- | ----- |
| Draft             | 작성 중  |
| Approved          | 승인 완료 |
| Rejected          | 반려    |
| Published         | 공개 완료 |
```

# 8. 문제 검수 및 승인 시스템

업로드된 문제는 관리자 또는 검수 권한이 있는 유저가 검토할 수 있다.

검수자는 다음 작업을 수행할 수 있다.

* 승인
* 반려
* 수정 요청
* 내부 메모 작성
* 난이도 조정 제안
* 태그 수정 제안

승인된 문제만 실제 문제 목록에 공개된다.


# 10. 유저 피드백 시스템

공개된 문제에 대해 유저는 피드백을 남길 수 있다.

피드백 기능:

* 댓글 작성
* 문제 오류 신고
* 테스트케이스 오류 신고
* 난이도 투표
* 태그 추천

유저 난이도 투표를 기반으로 실제 체감 난이도를 계산할 수 있다.

예시:

* "생각보다 어려움"
* "태그 부족"
* "예시 설명 부족"
